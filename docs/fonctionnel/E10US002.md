# E10US002 — Accès administrateur protégé (identifiant + mot de passe)

> Documentation fonctionnelle — **scénario de test pour un utilisateur non technique**.
> Objectif : au tout premier lancement, **définir** l'accès administrateur (identifiant + mot de
> passe) ; ensuite, **se connecter** pour accéder aux fonctions d'organisation (comme créer un
> tournoi) ; vérifier qu'sans connexion, ces fonctions sont **refusées**, alors que la simple
> **consultation** reste libre.
> Suivez les étapes dans l'ordre ; comparez à chaque fois ce que vous voyez au « résultat attendu ».

## Fonctionnalité concernée
L'application distingue deux usages : **consulter** (libre, sans compte) et **administrer**
(créer/configurer un tournoi). Les fonctions d'administration sont **protégées par un mot de passe**.
Il n'y a **qu'un seul accès administrateur** pour tout le poste (l'organisateur de la salle).

> ℹ️ **Où sont rangés l'identifiant et le mot de passe ?** Dans un petit fichier `.env` à côté de
> l'application, sur la machine qui fait serveur. C'est un choix **assumé** pour un outil de club en
> réseau local. Ce fichier sert aussi de **secours** : en cas d'oubli, la personne technique peut
> l'ouvrir (ou en effacer les deux lignes) pour que l'application **redemande** de définir l'accès.

## Pré-requis (comment y accéder)
- L'application est ouverte et indique **« En ligne »** en haut à droite (voir la fiche E00US010).
- L'encadré principal s'intitule **« Tournois »**.

## Scénario A — Tout premier lancement : définir l'accès

1. **Repérer la demande.** Au tout premier lancement, l'encadré **« Tournois »** affiche la rubrique
   **« Définir l'accès administrateur »** avec le message *« Premier lancement : choisissez
   l'identifiant et le mot de passe de l'organisateur. »*

2. **Choisir un identifiant.** Dans le champ **« Identifiant »**, saisissez un nom court
   (par ex. *« orga »*).

3. **Choisir un mot de passe.** Saisissez-le dans **« Mot de passe »**, puis **à l'identique** dans
   **« Confirmer le mot de passe »**.
   → *Si les deux ne correspondent pas :* un message rouge **« Les deux mots de passe ne
   correspondent pas. »** s'affiche et le bouton reste **inactif**.

4. **Valider.** Cliquez sur **« Définir l'accès »**.
   → *Résultat attendu :* vous êtes **immédiatement connecté** ; l'encadré affiche désormais la
   rubrique **« Nouveau tournoi »** (le formulaire de création). L'accès est enregistré : il ne
   sera plus **redemandé** aux prochains lancements (voir Scénario B).

## Scénario B — Lancements suivants : se connecter

5. **Repérer la connexion.** Tant que vous n'êtes pas connecté, l'encadré **« Tournois »** affiche la
   rubrique **« Connexion administrateur »**.

6. **Se connecter.** Saisissez l'**identifiant** et le **mot de passe** définis au premier lancement,
   puis cliquez sur **« Se connecter »**.
   → *Résultat attendu :* la rubrique **« Nouveau tournoi »** apparaît : vous pouvez créer un tournoi
   (voir la fiche E01US001).

7. **Se déconnecter (facultatif).** Sous le formulaire de création, le lien **« Se déconnecter
   (admin) »** ferme la session : l'encadré revient à **« Connexion administrateur »**.

## Ce qui reste possible sans être connecté (consultation)

8. **Voir les tournois existants.** Même **sans être connecté**, si des tournois ont déjà été créés,
   l'encadré affiche la liste **« Tournois existants »**. Cliquer sur une ligne **ouvre** le tournoi
   pour le consulter — la consultation ne demande **jamais** de mot de passe.

## Cas alternatifs & messages visibles

- **Mauvais identifiant ou mot de passe :** un **message rouge** s'affiche sous le formulaire
  (« Identifiant ou mot de passe invalide ») et vous **restez** sur l'écran de connexion.
- **Champs vides :** tant que l'identifiant ou le mot de passe est vide, le bouton reste **inactif**.
- **Créer un tournoi sans être connecté :** impossible — le formulaire de création n'apparaît pas
  tant que vous n'êtes pas connecté (à la place, l'écran de connexion s'affiche).
- **Session expirée après redémarrage du serveur :** si le poste serveur a redémarré, une création
  peut être refusée ; l'application vous **ramène** alors à l'écran de connexion — reconnectez-vous.
- **Mot de passe oublié :** demandez à la personne technique d'ouvrir le fichier `.env` sur la machine
  serveur (ou d'en effacer les deux lignes d'identifiants) ; au prochain accès, l'application
  **redemandera** de définir l'accès (Scénario A).

## Résultat final attendu
L'accès administrateur est **défini une fois** puis exigé à chaque session : sans connexion, créer un
tournoi est **refusé** ; connecté, on retrouve les fonctions d'administration. La **consultation**
reste libre, sans mot de passe.

---
*Critères d'acceptation couverts (E10US002) :* définition de l'accès au 1ᵉʳ usage ; connexion par
identifiant + mot de passe ouvrant une session ; fonctions admin (création de tournoi) refusées sans
session valide ; lecture publique inchangée.
